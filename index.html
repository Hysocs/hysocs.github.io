CobbleSpawnRegions

Create named regions that control Cobblemon natural spawns and custom Pokemon spawns.

Quick Start

  1. Run /csr giveclaimstick coords or /csr giveclaimstick chunk.
  2. Left-click a block with the stick to set point 1.
  3. Right-click a block with the stick to set point 2.
  4. Run /csr create <name> to save the selected region.
  5. Run /csr gui or /csr edit <regionId> to configure natural and custom spawn behavior.

Usage

Features

Region Control

  • Coords mode uses the exact two block positions selected with the claim stick.
  • Chunk mode expands the selected chunks into a full-height region from the world's bottom to top Y.
  • Priority decides which region controls overlapping positions.
  • Visualization toggles particle outlines for selected region bounds.
  • Region IDs are generated from names by lowercasing and replacing unsupported characters with underscores.
  • Config files live under config/cobblespawnregions, with regions saved in config/cobblespawnregions/regions.

Spawn Management

  • Natural restrictions affect wild Cobblemon spawns inside the controlling region.
  • Disable All blocks natural spawns in a region.
  • Blocked Species, Labels, and Conditions let you target specific natural spawn types.
  • Custom spawns are Pokemon the region spawns itself from selected Pokemon entries.
  • Spawn Timer is stored in ticks. 20 ticks equals 1 second.
  • Max Total Spawns caps all region-owned custom Pokemon. 0 means unlimited.
  • Pokemon entry settings include chance, levels, forms, aspects, spawn blocks, time, weather, catch rules, IVs, EVs, size, moves, and stay-in-region behavior.
  • Tracked spawns are saved in config/cobblespawnregions/tracked_spawns.json so unloaded entities still count against caps.

Pokemon Entry Settings

ChanceCompetitive entries are weighted together. Independent entries roll their own percent chance first.
LevelsEach spawn picks a level between the configured minimum and maximum.
Forms and AspectsForms and aspects are applied to the Cobblemon properties string when spawning.
Spawn BlocksUse block IDs like minecraft:grass_block, or #solid, #water, and #air.
Time and WeatherLimit entries to DAY, NIGHT, CLEAR, RAIN, THUNDER, or ALL.
Catch RulesMake entries uncatchable or restrict capture to selected Pokeball item IDs.
IVs, EVs, Size, MovesApply optional custom stats, scale, and initial move settings when spawned.
Stay In RegionCustom-spawned Pokemon can path back toward a random, center, or closest region point.

JSON Config Guide

Before Editing JSON

  • Region files live in config/cobblespawnregions/regions.
  • Each region is one region_name.jsonc file.
  • Pokemon go inside the selectedPokemon array.
  • Objects use { }. Lists use [ ]. Text must use quotes.
  • Put a comma between fields and between Pokemon entries, but not after the last item.
  • Run /csr reload after editing files.

Common Values

  • spawnTime: ALL, DAY, or NIGHT.
  • spawnWeather: ALL, CLEAR, RAIN, or THUNDER.
  • spawnChanceType: COMPETITIVE or INDEPENDENT.
  • allowedBlocks: #solid, #water, #air, or block IDs.
  • returnTarget: RANDOM, CENTER, or CLOSEST.
  • Most caps use 0 as unlimited.

CobbleBattleRewards

Config-driven rewards for Cobblemon battles, captures, losses, and forfeits.

Quick Start

  1. Start the server once so config/cobblebattlerewards/config.jsonc is generated.
  2. Edit one reward map: battleWonRewards, battleLostRewards, battleForfeitRewards, or captureRewards.
  3. Add a named reward object, such as "money_100": { ... }.
  4. Choose type: command or item.
  5. Run /cbr reload and test the trigger in-game.

Usage

Features

Reward Triggers

  • battleWonRewards runs when a player wins a battle.
  • battleLostRewards runs when a player loses a battle.
  • battleForfeitRewards runs when a player forfeits or flees.
  • captureRewards runs when a Pokemon is captured.
  • battleTypes can limit rewards to wild, npc, or pvp.
  • allowedDimensions can limit rewards to dimensions such as minecraft:overworld.

Reward Filters

  • conditions match the opponent Pokemon. Empty means any Pokemon.
  • conditionsBlacklist reverses condition logic.
  • minLevel and maxLevel restrict by opponent level.
  • chance is a percent roll from 0 to 100.
  • cooldown is per player and reward, in seconds.
  • order runs lower numbers first, letting rare rewards block common ones.
  • excludedRewards prevents named lower-priority rewards from also being paid.

What You Can Use

Reward Typescommand and item.
Battle Typeswild, npc, and pvp.
TriggersBattleWon, BattleLost, BattleForfeit, and Captured.
Inventory Fulldrop or skip.
Species ConditionUse resource IDs like cobblemon:pikachu.
Property ConditionUse key/value checks like type=electric or shiny=true.
AND GroupsUse nested lists: ["type=electric", "shiny=true"].
MiniMessageMessages support tags such as <green> and <gold>.

Config Usage

JSON Rules For This Mod

  • All rewards are edited in config/cobblebattlerewards/config.jsonc.
  • Reward maps are objects: "reward_name": { ... }.
  • Every reward name in the same map must be unique.
  • Text must use quotes. Lists use [ ]. Objects use { }.
  • Use commas between rewards and fields, but not after the last item.
  • Run /cbr reload after changes.

Placeholders

  • %player%: player name.
  • %pokemon%: opponent Pokemon species.
  • %level%: opponent Pokemon level.
  • %battleType%: wild, npc, or pvp.
  • %coords%: player X,Y,Z.
  • %trigger%: reward trigger.
  • %dimension%: player dimension.
  • %time%: cooldown time left, only for cooldown messages.

What Not To Use

  • Do not use reward type names other than command or item.
  • Do not include a leading slash in command.
  • Do not put plain item IDs in itemStack; it must be an escaped ItemStack JSON string.
  • Do not use battle types outside wild, npc, or pvp.
  • Do not duplicate reward names inside the same reward map.
