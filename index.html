CobbleSpawnRegions

Create named regions that control Cobblemon natural spawns and custom Pokemon spawns.

View on Modrinth Commands
Mod ID
cobblespawnregions
Version
1.0.1 Checking Modrinth...
Requires

Modrinth
Open project page

Quick Start

  1. Run /csr giveclaimstick coords or /csr giveclaimstick chunk.
  2. Left-click a block with the stick to set point 1.
  3. Right-click a block with the stick to set point 2.
  4. Run /csr create <name> to save the selected region.
  5. Run /csr gui or /csr edit <regionId> to configure natural and custom spawn behavior.

claiming GIF unavailable

Usage

Features

Region Control

  • Coords mode uses the exact two block positions selected with the claim stick.
  • Chunk mode expands the selected chunks into a full-height region from the world's bottom to top Y.
  • Priority decides which region controls overlapping positions.
  • Visualization toggles particle outlines for selected region bounds.
  • Region IDs are generated from names by lowercasing and replacing unsupported characters with underscores.
  • Config files live under config/cobblespawnregions, with regions saved in config/cobblespawnregions/regions.

Spawn Management

  • Natural restrictions affect wild Cobblemon spawns inside the controlling region.
  • Disable All blocks natural spawns in a region.
  • Blocked Species, Labels, and Conditions let you target specific natural spawn types.
  • Custom spawns are Pokemon the region spawns itself from selected Pokemon entries.
  • Spawn Timer is stored in ticks. 20 ticks equals 1 second.
  • Max Total Spawns caps all region-owned custom Pokemon. 0 means unlimited.
  • Pokemon entry settings include chance, levels, forms, aspects, spawn blocks, time, weather, catch rules, IVs, EVs, size, moves, and stay-in-region behavior.
  • Tracked spawns are saved in config/cobblespawnregions/tracked_spawns.json so unloaded entities still count against caps.

Pokemon Entry Settings

ChanceCompetitive entries are weighted together. Independent entries roll their own percent chance first.
LevelsEach spawn picks a level between the configured minimum and maximum.
Forms and AspectsForms and aspects are applied to the Cobblemon properties string when spawning.
Spawn BlocksUse block IDs like minecraft:grass_block, or #solid, #water, and #air.
Time and WeatherLimit entries to DAY, NIGHT, CLEAR, RAIN, THUNDER, or ALL.
Catch RulesMake entries uncatchable or restrict capture to selected Pokeball item IDs.
IVs, EVs, Size, MovesApply optional custom stats, scale, and initial move settings when spawned.
Stay In RegionCustom-spawned Pokemon can path back toward a random, center, or closest region point.

Config Guide

JSON Rules For This Mod

  • Region files live in config/cobblespawnregions/regions.
  • Each region is one region_name.jsonc file.
  • Pokemon go inside the selectedPokemon array.
  • Objects use { }. Lists use [ ]. Text must use quotes.
  • Put a comma between fields and between Pokemon entries, but not after the last item.
  • Run /csr reload after saving changes.

Common Values

  • spawnTime: ALL, DAY, or NIGHT.
  • spawnWeather: ALL, CLEAR, RAIN, or THUNDER.
  • spawnChanceType: COMPETITIVE or INDEPENDENT.
  • allowedBlocks: #solid, #water, #air, or block IDs.
  • returnTarget: RANDOM, CENTER, or CLOSEST.
  • Most caps use 0 as unlimited.

CobbleBattleRewards

Config-driven rewards for Cobblemon battles, captures, losses, and forfeits.

Modrinth
Open project page

Quick Start

  1. Start the server once to generate config/cobblebattlerewards/config.jsonc.
  2. Edit one reward map: battleWonRewards, battleLostRewards, battleForfeitRewards, or captureRewards.
  3. Add a named reward object, such as "money_100": { ... }.
  4. Choose type: command or item.
  5. Run /cbr reload and test the trigger in-game.

Usage

Features

Reward Triggers

  • battleWonRewards runs when a player wins a battle.
  • battleLostRewards runs when a player loses a battle.
  • battleForfeitRewards runs when a player forfeits or flees.
  • captureRewards runs when a Pokemon is captured.
  • battleTypes can limit rewards to wild, npc, or pvp.
  • allowedDimensions can limit rewards to dimensions such as minecraft:overworld.

Reward Filters

  • conditions match the opponent Pokemon. Empty means any Pokemon.
  • conditionsBlacklist reverses condition logic.
  • minLevel and maxLevel restrict by opponent level.
  • chance is a percent roll from 0 to 100.
  • cooldown is per player and reward, in seconds.
  • order runs lower numbers first, letting rare rewards block common ones.
  • excludedRewards prevents named lower-priority rewards from also being paid.

What You Can Use

Reward Typescommand and item.
Battle Typeswild, npc, and pvp.
TriggersBattleWon, BattleLost, BattleForfeit, and Captured.
Inventory Fulldrop or skip.
Species ConditionUse resource IDs like cobblemon:pikachu.
Property ConditionUse key/value checks like type=electric or shiny=true.
AND GroupsUse nested lists: ["type=electric", "shiny=true"].
MiniMessageMessages support tags such as <green> and <gold>.

Config Usage

JSON Rules For This Mod

  • All rewards are edited in config/cobblebattlerewards/config.jsonc.
  • Reward maps are objects: "reward_name": { ... }.
  • Every reward name in the same map must be unique.
  • Text must use quotes. Lists use [ ]. Objects use { }.
  • Use commas between rewards and fields, but not after the last item.
  • Run /cbr reload after saving changes.

Placeholders

  • %player%: player name.
  • %pokemon%: opponent Pokemon species.
  • %level%: opponent Pokemon level.
  • %battleType%: wild, npc, or pvp.
  • %coords%: player X,Y,Z.
  • %trigger%: reward trigger.
  • %dimension%: player dimension.
  • %time%: cooldown time left, only for cooldown messages.

What Not To Use

  • Do not use reward type names other than command or item.
  • Do not include a leading slash in command.
  • Do not put plain item IDs in itemStack; it must be an escaped ItemStack JSON string.
  • Do not use battle types outside wild, npc, or pvp.
  • Do not duplicate reward names inside the same reward map.

CobbleHunts

Pokemon hunt missions for Cobblemon with global hunts, solo difficulty tiers, rewards, leaderboards, and a tracking brush.

Quick Start

  1. Start the server once to generate the config/cobblehunts folder.
  2. Run /hunts to open the player hunt GUI.
  3. Use global hunts or start a solo mission from the enabled difficulty tiers.
  4. Catch a matching Pokemon, then turn it in from the hunt GUI.
  5. Use /hunts editor to edit hunts in-game or edit the JSON files directly.

Hunts GUI

hunts GUI screenshot or GIF unavailable

Usage

Features

Hunt Types

  • Global hunts are shared server hunts with one shared timer.
  • Solo hunts are per-player missions with easy, normal, medium, and hard tiers.
  • Global active count controls how many global Pokemon are shown at once.
  • Cooldowns apply after hunts expire or complete.
  • Auto accept can start solo hunts automatically when a player is ready.

Turn-In Flow

  • Players catch a Pokemon that matches an active hunt.
  • The mod can require the Pokemon to be captured after the hunt started.
  • Turn-ins can remove the Pokemon from the player's storage.
  • Admins can revert a turn-in from the recent removed Pokemon cache.
  • Capture messages can tell players when a caught Pokemon matches an active hunt.

What A Hunt Can Require

SpeciesUse Cobblemon species names such as pikachu.
FormUse a form name such as Normal, or leave it null.
AspectsRequire one or more Cobblemon aspects in a list.
ChanceWeights how often the entry is selected from its pool.
GenderUse male, female, genderless, or random.
NatureUse a nature path such as adamant or timid.
IV RangeA number above 0 picks that many required IV stats at 20+ IVs.
RewardsUse command or item rewards from the matching loot pool.

Config Usage

Config Files

  • config/cobblehunts/config.jsonc controls timers, toggles, messages, permissions, and tracking brush settings.
  • config/cobblehunts/pokemon_pools.jsonc controls which Pokemon can appear in each hunt pool.
  • config/cobblehunts/loot_pools.jsonc controls rewards for global and solo tiers.
  • Run /hunts reload after saving changes.

Permission Nodes

  • cobblehunts.hunts: open the hunt GUI.
  • cobblehunts.global: access global hunts.
  • cobblehunts.solo.easy, .normal, .medium, .hard: access solo tiers.
  • cobblehunts.givebrush: give tracking brushes.
  • Default permission and op levels are both 2.

Reward Modes

  • weight picks one reward using each reward's chance as a weight.
  • percentage rolls every reward independently using chance as a 0.0 to 1.0 probability.
  • Do not use a leading slash in command rewards.
  • Use escaped ItemStack JSON strings for itemStackString.

Quick Start

  1. Start the server once to generate config/blanketrtp/config.jsonc.
  2. Run /rtp to teleport to a random safe location.
  3. Run /rtp no_glide if you want a normal teleport without glide mode.
  4. Run /rtp biome minecraft:plains to search for a specific biome.
  5. Edit cooldowns, redirects, dimension rules, and gliding behavior in the config, then run /rtp reload to apply changes.

Usage

Features

Teleport Control

  • Random RTP searches inside the configured radius.
  • Biome RTP scans for a requested biome with configurable attempt and tick limits.
  • Dimension redirects can send Nether or End RTP requests back to the Overworld.
  • Dimension safety rules support auto, surface, cave, and floating modes.
  • Slow loading prepares chunks before releasing the player.

Player Experience

  • Gliding can teleport players above the target and let them glide down.
  • Glider item can temporarily equip a configured item such as an elytra.
  • Particles can trail the player during glide mode.
  • Cooldown tiers allow VIP/member/default cooldowns through permissions.
  • Messages use MiniMessage-style text and placeholders.

Reference

Commands

/rtp
blanketrtp.rtp

Runs a normal random teleport. Default permission level and op level are 0.

/rtp no_glide
blanketrtp.rtp.noglide

Runs RTP with glide mode disabled for that teleport. Default permission level and op level are 0.

/rtp biome <biome>
blanketrtp.rtp.biome

Searches for the requested biome, such as minecraft:plains. Default permission level and op level are 0.

/rtp biome <biome> no_glide
blanketrtp.rtp.biome

Biome RTP with glide mode disabled for that teleport.

/rtpbiome <biome>
blanketrtp.rtp.biome

Optional standalone biome command. It only exists when enableBiomeStandalone is true.

/rtp reload
blanketrtp.admin.reload

Reloads the BlanketRTP config. Default permission level and op level are 2.

/rtp stopgliding <player>
blanketrtp.admin.stopgliding

Forces a player out of RTP glide mode. Default permission level and op level are 2.

Config Usage

JSON Rules For This Mod

  • Main config lives at config/blanketrtp/config.jsonc.
  • Dimension IDs use full IDs such as minecraft:overworld.
  • Biome IDs use full IDs such as minecraft:plains.
  • Cooldown durations, slow load times, and biome tick limits are numbers.
  • Run /rtp reload after saving changes.

Placeholders

  • {time}: cooldown seconds left.
  • {biome}: requested biome ID.
  • {x}, {y}, {z}: destination coordinates.
  • {percent}: slow-load progress percentage.
