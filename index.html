CobbleSpawnRegions

Create named regions that control Cobblemon natural spawns and custom Pokemon spawns.

Mod ID
cobblespawnregions
Version
1.0.1
Requires
Fabric Loader, Fabric API, Fabric Language Kotlin, Cobblemon

Quick Start

  1. Run /csr giveclaimstick coords or /csr giveclaimstick chunk.
  2. Left-click a block with the stick to set point 1.
  3. Right-click a block with the stick to set point 2.
  4. Run /csr create <name> to save the selected region.
  5. Run /csr gui or /csr edit <regionId> to configure natural and custom spawn behavior.

Region Modes

Coords mode uses the exact two block positions selected with the claim stick.

Chunk mode expands the selected chunks into a full-height region from the world's bottom to top Y.

Region IDs are generated from the name by lowercasing it and replacing unsupported characters with underscores.

Core Commands

/csr giveclaimstick coords|chunkGives a region selection stick.
/csr create <name>Creates a region from the current two-point selection.
/csr listLists regions by priority order.
/csr guiOpens the region list GUI.
/csr edit <regionId>Opens the editor for one region.
/csr visualize <regionId>Toggles particle visualization for the region bounds.
/csr checkShows which region controls your current position and active natural restrictions.
/csr info <regionId>Prints bounds, priority, spawn settings, caps, and restriction counts.
/csr teleport <regionId>Teleports to a region. /csr tp is an alias.
/csr delete <regionId>Deletes a region and clears its tracked custom spawns.
/csr reloadReloads config files and rescans loaded chunks.

Natural Spawn Controls

Natural spawn settings affect wild Cobblemon spawns inside the region that wins priority at a position.

  • Disable All blocks natural spawns in the region.
  • Exclude Owned Pokemon lets owned Pokemon bypass region restrictions.
  • Blocked Species blocks selected species.
  • Excluded Labels blocks spawns matching Cobblemon labels.
  • Conditions blocks spawns matching configured condition strings.

Custom Spawn Controls

Custom spawns are Pokemon the region spawns itself. They obey the region timer, selected Pokemon entries, spawn caps, block filters, time, weather, and region priority.

  • Spawn Timer is stored in ticks. 20 ticks equals 1 second.
  • Spawn Amount is the number of spawn attempts each timer cycle.
  • Max Total Spawns caps all region-owned custom Pokemon. 0 means unlimited.
  • Require Player Nearby only spawns while a player is within the configured range around the bounds.

Pokemon Entry Settings

ChanceCompetitive entries are weighted against each other. Independent entries roll their own percent chance first.
LevelsEach spawn picks a level between the configured minimum and maximum.
Forms and AspectsForms and aspects are applied to the Cobblemon properties string when spawning.
Spawn BlocksAllow literal block IDs like minecraft:grass_block, or tags #solid, #water, and #air. Empty means any scanned floor.
Time and WeatherEntries can be limited to DAY, NIGHT, CLEAR, RAIN, THUNDER, or ALL.
Catch RulesEntries can be uncatchable or limited to selected Pokeball item IDs.
IVs, EVs, Size, MovesOptional per-entry overrides are applied when the Pokemon is spawned.
Stay In RegionCustom-spawned Pokemon can path back toward a random, center, or closest region point.

Priority and Overlap

When regions overlap, the highest priority region controls that position. If priorities match, smaller regions win before larger regions, then region ID is used as a final tie-breaker.

Use /csr priority <regionId> <value> to change priority.

Config Files

Main config lives at config/cobblespawnregions. Regions are saved as JSON/JSONC files under config/cobblespawnregions/regions.

The mod also tracks custom spawned Pokemon in config/cobblespawnregions/tracked_spawns.json so unloaded entities still count against caps.

Admin Commands

/csr addmon <regionId> <pokemon> [form] [aspects]Adds a custom spawn entry.
/csr removemon <regionId> <pokemon> [form] [aspects]Removes a custom spawn entry.
/csr forcespawn <regionId> [amount]Immediately attempts custom spawns.
/csr killspawned <regionId> [loaded|tracked]Removes region-managed spawned Pokemon.
/csr setspawnamount <regionId> <amount>Changes attempts per timer cycle.
/csr playeractivation <regionId> true|false [range]Requires or disables nearby-player activation.
/csr forcechunks <regionId> true|false [maxChunks] [radius]Toggles force-loaded chunks for a region.
/csr clone <sourceRegionId> <newName>Copies settings from one region onto the current selection.
/csr rename <regionId> <newName>Changes the display name.
/csr inspectnearestPrints details about the nearest Pokemon and extracted conditions.
